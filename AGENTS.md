# AI Pet Name Generator

## 项目介绍

这是一个从 `temp/landing.html` 迁移而来的 Next.js 单页宠物命名工具。页面通过服务端输出 SEO 内容，并在客户端提供宠物类型、性别、命名风格、关键词、生成、重置和复制等交互。`temp/` 是视觉与内容基准，不是产品运行代码。

## 前端目录结构

- `app/`：App Router 路由、全局样式和元数据。
- `app/page.tsx`：产品首页组合入口。
- `app/design-system/page.tsx`：仅开发环境可访问的设计系统预览。
- `components/ui/`：无业务语义的基础组件（Button、Chip、Card、Container、SectionHeading）。
- `components/landing/`：首页业务组件与生成器状态。
- `public/`：从原型复用的静态品牌资源。
- `temp/`：原生 HTML、品牌规范和原始素材，仅作迁移参考。

## 设计系统架构

设计变量集中在 `app/globals.css` 的 `:root`：颜色、字体、字号、行高、间距、圆角、阴影、焦点和容器宽度均通过 token 使用。基础组件只依赖这些 token；页面组件通过组合基础组件完成布局。桌面、平板（≤920px）与手机（≤560px）响应规则也集中维护。

## 组件复用规范

开发页面时，必须优先复用已有组件。如果已有组件能通过 `props`、`variant` 或 `className` 扩展，应优先扩展，不得重新创建相似组件。只有现有组件在语义、交互或结构上确实无法满足需求时才能新增组件。新增通用组件放入 `components/ui/`；包含具体业务语义的组件放入对应业务目录。禁止在页面中重复堆叠可抽象的样式或复制 token 数值。

## 后续开发注意事项

- 保持按钮、输入和可点击控件的键盘焦点、禁用和按压状态完整。
- 新颜色、间距、圆角或阴影先添加为设计 token，再在组件中使用。
- 组件变体应保持 API 小而明确；业务数据与视觉组件分离。
- 修改响应布局时同时验证 1180px、920px 和 560px 三个边界。
- `/design-system` 仅用于本地开发验证，生产环境必须返回 404。
- `temp/landing.html` 是像素还原的参考基准；产品实现以 React/Next.js 组件为唯一运行来源。
